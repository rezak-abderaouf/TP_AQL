# TP1 Unit Tests

## Bug Reports
* **Exercice 1 (Palindrome):** A bug was found in the `isPalindrome` method. Inside the while loop, the increment/decrement operators were reversed (`j++` and `i--`), causing a `StringIndexOutOfBoundsException`. The fix is proposed in the `Exo1Correction` class where the pointers correctly converge (`i++` and `j--`).
* **Note on Coverage Criteria (Exercice 1):** The tests written for Line Coverage also achieve 100% Branch Coverage and 100% Condition Coverage. Because the `isPalindrome` method contains no compound conditions (`&&` or `||`), the basic paths automatically satisfy the stricter coverage criteria. The test classes have been duplicated in their respective packages to respect the repository nomenclature.
* **Exercice 2 (Anagram):** Multiple bugs were found. First, syntax errors in the provided PDF code (`!` instead of `!=`). Second, an off-by-one logic error in the `for` loop condition (`i <= s1.length()`) caused a `StringIndexOutOfBoundsException`. The loop was fixed to `i < s1.length()` in `Exo2Correction`.
* **Note on Coverage Criteria (Exercice 2):** By adding a test for a null second string, the Line Coverage test suite achieved 100% Branch and Condition Coverage simultaneously. The test classes were duplicated into their respective folders.
* **Exercice 3 (Binary Search):** A logical bug was found in the while loop condition. Using `while (low < high)` causes the search to fail when the target element is at the very edges of the array or if the array has only one element. The condition was corrected to `while (low <= high)` in `Exo3Correction`.
* **Note on Coverage Criteria (Exercice 3):** The tests written for Line Coverage automatically achieve 100% Branch and Condition coverage as there are no compound boolean conditions. The files were duplicated to respect the repository structure.
* **Exercice 4 (QuadraticEquation):** The provided code contained multiple syntax errors (`double[` instead of `double[]`, missing operators like `2*a`, and misplaced brackets causing unconditional exceptions). These were fixed in `Exo4Correction`.
* **Note on Coverage Criteria (Exercice 4):** Since the method only uses simple `if` conditions without compound logical operators, the Line Coverage test suite naturally achieved 100% Branch and Condition Coverage.
* **Exercice 5 (RomanNumeral):** Two bugs were found. An `ArrayIndexOutOfBoundsException` caused by `i <= symbols.length` in the `for` loop, and a logic error where `while (n > values[i])` failed to catch exact numeral matches. These were fixed to `<` and `>=` respectively in `Exo5Correction`.
* **Note on Coverage Criteria:** Testing both boundaries of the `||` operator (`0` and `4000`) achieved 100% Condition Coverage concurrently with Line Coverage.
* **Exercice 6 (FizzBuzz):** A logical bug was found in the initial bounds check. The condition `if (n <= 1)` incorrectly threw an exception for the positive number `1`. This was fixed to `if (n <= 0)` in `Exo6Correction` to align with the requirement that `n` must be positive.
* **Note on Coverage Criteria (Exercice 6):** Because the method consists solely of independent, simple `if` statements without compound operators, the Line Coverage tests natively achieve 100% Branch and Condition Coverage.
